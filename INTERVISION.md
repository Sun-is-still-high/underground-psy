## 1. Краткое описание изменений

Интервизии переходят от полностью управляемых админом к **самоорганизуемым психологами**. Появляется два формата: постоянная группа (с регулярным расписанием) и одноразовая интервизия. Видеосвязь через встроенный Jitsi. Подтверждение сессии — по логике троек (все подтверждают).

### Что меняется

| Было | Стало |
|------|-------|
| Группы создаёт только админ | Психолог подаёт заявку → админ одобряет |
| Сессии создаёт только админ | Сессии генерируются автоматически по расписанию + создатель может создавать вручную |
| Посещаемость отмечает админ | Участники сами подтверждают, что сессия состоялась |
| Нет одноразовых интервизий | Психолог может создать одноразовую интервизию |
| Нет самозаписи в группу | Психологи записываются в группу сами |
| Внешняя ссылка на встречу | Встроенный Jitsi |
| Психолог не видит расписание | Расписание видно на дашборде |
| Нет напоминаний | Напоминания за 24ч и 1ч |
| Статус сессии меняется вручную | Автоматический переход по времени |
| can_consult пересчитывается при сохранении посещаемости | Динамическое вычисление при запросе |

## 2. Два формата интервизий

### 2.1 Постоянная группа

Долгосрочное объединение психологов с регулярным расписанием.

**Создание:**
1. Психолог подаёт заявку на создание группы: название, описание, расписание (день недели + время), максимум участников (до 6)
2. Админ одобряет или отклоняет заявку
3. Одобрено → группа создаётся, сессии автоматически генерируются на месяц вперёд
4. Создатель становится участником и получает права администратора группы

**Автогенерация сессий:**
- Система генерирует сессии по расписанию на 1 месяц вперёд
- Пересчёт (догенерация) — ежедневно через Scheduler, чтобы всегда был месяц сессий впереди
- Тема сессии — не обязательна. Создатель может задать тему позже или она определяется на встрече
- Создатель может изменить расписание → старые незавершённые сессии пересоздаются

**Участники:**
- Самозапись: психолог видит группу → записывается автоматически (если есть места)
- Создатель видит, кто записался, и может отклонить/исключить участника
- Максимум 6 участников (включая создателя)

**Права создателя группы:**
- Создавать / отменять сессии
- Менять расписание
- Исключать участников
- Отклонять записавшихся
- Закрыть группу (все запланированные сессии отменяются, участники получают уведомление)
- Задавать / менять тему сессии

**Закрытие группы:**
- Создатель или админ может закрыть группу
- Все запланированные сессии со статусом SCHEDULED автоматически отменяются
- Все участники получают уведомление (колокольчик + email)

### 2.2 Одноразовая интервизия

Разовая встреча, не привязанная к постоянной группе. Механика аналогична слотам в тройках.

**Создание:**
1. Психолог создаёт одноразовую интервизию: дата, время, длительность, тема (не обязательна)
2. Появляется в ленте доступных интервизий
3. Другие психологи записываются

**Ограничения:**
- Минимум 4 участника (включая создателя)
- Максимум 6 участников
- Дедлайн записи — за 1 час до начала
- Если к дедлайну записалось меньше 4 — отменяется автоматически

**Права создателя одноразовой интервизии:**
- Отменить интервизию (участники получают уведомление)
- Задать / изменить тему

### 2.3 Общее для обоих форматов

- Видеосвязь через встроенный Jitsi
- Подтверждение после сессии — все участники подтверждают. Если хотя бы один не подтвердил — не засчитывается никому
- Засчитывается одинаково для `can_consult`
- Блокировка пересечений по времени (как в тройках)

## 3. User Stories

### US-1: Заявка на создание постоянной группы

> Как психолог, я хочу подать заявку на создание интервизионной группы, чтобы организовать регулярные встречи с коллегами.

**Критерии приёмки:**
- Психолог заполняет: название, описание, день недели + время, максимум участников (4–6)
- Заявка отправляется на модерацию
- Психолог видит статус заявки (pending / approved / rejected)
- Одобрена → группа создаётся, сессии генерируются на месяц вперёд
- Отклонена → психолог видит причину

### US-2: Самозапись в постоянную группу

> Как психолог, я хочу записаться в интервизионную группу, чтобы участвовать в регулярных встречах.

**Критерии приёмки:**
- Психолог видит список групп с открытыми местами
- Записывается автоматически (если есть места)
- Создатель группы видит нового участника и может исключить
- Блокировка записи при конфликте по времени с другими слотами/сессиями

### US-3: Создание одноразовой интервизии

> Как психолог, я хочу создать разовую интервизию, чтобы обсудить случай с коллегами без привязки к постоянной группе.

**Критерии приёмки:**
- Психолог задаёт: дату, время, длительность, тему (опционально)
- Одноразовая интервизия появляется в ленте
- Другие психологи записываются (до 6 включая создателя)
- Минимум 4 участника для проведения
- Дедлайн записи — за 1 час до начала
- Меньше 4 к дедлайну → автоматическая отмена с уведомлением

### US-4: Проведение сессии (общая для обоих форматов)

> Как психолог, я хочу подключиться к видеовстрече и провести интервизию.

**Критерии приёмки:**
- Кнопка «Подключиться» доступна за 5 минут до начала
- Подключение первого участника → статус IN_PROGRESS
- Видеосвязь через встроенный Jitsi (до 6 участников)
- Уведомление о скором завершении по истечении длительности. Автоотключения нет
- После сессии — все участники подтверждают, что сессия состоялась
- Все подтвердили → засчитывается, счётчик +1 каждому
- Хотя бы один не подтвердил → не засчитывается никому

### US-5: Управление группой (создатель)

> Как создатель группы, я хочу управлять составом и расписанием.

**Критерии приёмки:**
- Создавать / отменять отдельные сессии
- Менять расписание (день/время) → пересоздание будущих сессий
- Исключать участников
- Отклонять записавшихся
- Задавать / менять тему сессии
- Закрыть группу → отмена всех запланированных сессий + уведомление участникам

### US-6: Расписание на дашборде

> Как психолог, я хочу видеть расписание моих предстоящих интервизий.

**Критерии приёмки:**
- На дашборде: список предстоящих сессий (дата, время, тема, группа/одноразовая)
- Прогресс: «Вы посетили X из 3 обязательных интервизий за последние 30 дней»
- Ссылка для подключения (за 5 минут до начала)

### US-7: Напоминания

> Как психолог, я хочу получать напоминания о предстоящих интервизиях, чтобы не пропустить.

**Критерии приёмки:**
- Уведомление за 24 часа до сессии (колокольчик + email)
- Уведомление за 1 час до сессии (колокольчик + email)

### US-8: Модерация заявок (админ)

> Как админ, я хочу одобрять или отклонять заявки на создание групп.

**Критерии приёмки:**
- Список заявок со статусом pending
- Одобрить → группа создаётся
- Отклонить с комментарием → психолог получает уведомление
- Админ может закрыть любую группу
- Админ может отменить любую сессию

### US-9: Настройки платформы (админ)

> Как админ, я хочу управлять глобальными настройками интервизий через интерфейс.

**Критерии приёмки:**
- Страница «Настройки платформы» в админке
- Изменение `min_sessions` (минимум интервизий за 30 дней для can_consult)
- Другие глобальные параметры по мере необходимости

## 4. Бизнес-правила

### Интервизионная группа

- Создаётся по заявке психолога → одобрение админа
- Минимум участников: 4, максимум: 6 (включая создателя)
- Расписание: день недели + время. Сессии автогенерируются на 1 месяц вперёд
- Создатель — администратор группы с полными правами
- Самозапись автоматическая (без одобрения), но создатель может исключить
- Закрытие группы отменяет все запланированные сессии

### Одноразовая интервизия

- Создаётся любым психологом, без одобрения
- Минимум 4, максимум 6 участников
- Дедлайн записи: за 1 час до начала
- Меньше 4 к дедлайну → автоматическая отмена
- Создатель имеет права: отмена, изменение темы

### Проведение сессии

- Подключение за 5 минут до начала
- Видеосвязь: Jitsi (встроенный), до 6 участников
- Автоматический переход статусов: SCHEDULED → IN_PROGRESS (первое подключение) → COMPLETED (все подтвердили)
- Уведомление о скором завершении, автоотключения нет
- Подтверждение: все участники должны подтвердить. Один не подтвердил → не засчитывается никому
- Блокировка пересечений по времени с другими слотами (тройки, интервизии)

### can_consult

- Rolling window: 30 дней
- Минимум: настраивается в `intervision_settings.min_sessions` (по умолчанию 3)
- Вычисляется **динамически** при запросе каталога (не хранится как поле)
- Одноразовые и групповые интервизии засчитываются одинаково
- При `can_consult = false`: психолог скрыт из каталога, новые записи невозможны, существующие сохраняются

### Тема сессии

- Не обязательна при создании
- Может быть задана или изменена создателем в любой момент до начала
- Если не задана — определяется участниками на встрече

## 6. State Machines

### Intervision Group

```
[pending] → админ одобрил → [active]
[pending] → админ отклонил → (заявка отклонена, группа не создаётся / остаётся pending с rejection_comment)
[active] → создатель или админ закрыл → [closed]
[closed] — конечное состояние
```

### Intervision Session

```
[scheduled] → первый участник подключился → [in_progress]
[scheduled] → дедлайн, меньше 4 участников (одноразовая) → [cancelled]
[scheduled] → создатель/админ отменил → [cancelled]
[in_progress] → все участники подтвердили → [completed]
[cancelled] — конечное состояние
[completed] — конечное состояние
```

## 7. API-контракты

### 7.1 Интервизионные группы

#### POST /api/intervision-groups
Подать заявку на создание группы.

- **Auth:** required (role: PSYCHOLOGIST, status: active)
- **Body:**
```json
{
  "name": "Группа по работе с тревогой",
  "description": "Еженедельный разбор случаев...",
  "schedule_day": "tuesday",
  "schedule_time": "19:00",
  "duration_minutes": 60,
  "max_participants": 6
}
```
- **Response 201:** `{ "id": 1, "status": "pending" }`

#### GET /api/intervision-groups
Список активных групп (с информацией о свободных местах).

- **Auth:** required (role: PSYCHOLOGIST)
- **Response 200:**
```json
{
  "data": [
    {
      "id": 1,
      "name": "Группа по работе с тревогой",
      "description": "...",
      "creator": { "id": 1, "name": "Анна" },
      "schedule_day": "tuesday",
      "schedule_time": "19:00",
      "duration_minutes": 60,
      "max_participants": 6,
      "current_participants": 4,
      "is_member": false
    }
  ]
}
```

#### POST /api/intervision-groups/{id}/join
Записаться в группу.

- **Auth:** required (role: PSYCHOLOGIST)
- **Response 201:** `{ "joined": true }`
- **Response 409:** `{ "error": "Группа заполнена" }`
- **Response 409:** `{ "error": "Конфликт по времени" }`

#### DELETE /api/intervision-groups/{id}/leave
Покинуть группу.

- **Auth:** required (участник группы, не создатель)
- **Response 200:** `{ "left": true }`

#### PUT /api/intervision-groups/{id}
Редактировать группу (создатель).

- **Auth:** required (только создатель)
- **Body:** `{ "schedule_day?": "...", "schedule_time?": "...", "name?": "...", "description?": "..." }`
- **Response 200:** `{ ... }`
- **Side effect:** при изменении расписания — пересоздание будущих сессий со статусом scheduled

#### DELETE /api/intervision-groups/{id}
Закрыть группу (создатель или админ).

- **Auth:** required (создатель или ADMIN)
- **Response 200:** `{ "status": "closed" }`
- **Side effect:** все scheduled-сессии → cancelled, уведомление участникам

#### DELETE /api/intervision-groups/{id}/participants/{userId}
Исключить участника (создатель).

- **Auth:** required (только создатель)
- **Response 200:** `{ "removed": true }`
- **Side effect:** уведомление исключённому

### 7.2 Одноразовые интервизии

#### POST /api/intervision-sessions
Создать одноразовую интервизию.

- **Auth:** required (role: PSYCHOLOGIST, status: active)
- **Body:**
```json
{
  "starts_at": "2026-03-10T19:00:00Z",
  "duration_minutes": 60,
  "topic": "Разбор случая с ПТСР",
  "max_participants": 6
}
```
- **Response 201:** `{ "id": 1, "type": "one_time", "status": "scheduled" }`
- **Response 409:** `{ "error": "Конфликт по времени" }`

#### GET /api/intervision-sessions
Лента доступных интервизий (одноразовых + предстоящие сессии открытых групп).

- **Auth:** required (role: PSYCHOLOGIST)
- **Query:** `?type=one_time|recurring&date_from=...&date_to=...&page=1`
- **Response 200:**
```json
{
  "data": [
    {
      "id": 1,
      "type": "one_time",
      "topic": "Разбор случая с ПТСР",
      "starts_at": "2026-03-10T19:00:00Z",
      "duration_minutes": 60,
      "creator": { "id": 1, "name": "Борис" },
      "max_participants": 6,
      "current_participants": 3,
      "status": "scheduled"
    }
  ]
}
```

#### POST /api/intervision-sessions/{id}/join
Записаться на одноразовую интервизию.

- **Auth:** required (role: PSYCHOLOGIST)
- **Response 201:** `{ "joined": true }`
- **Response 409:** `{ "error": "Максимум участников" }`
- **Response 409:** `{ "error": "Конфликт по времени" }`

## 10. can_consult — динамическое вычисление

### Логика

`can_consult` **больше не хранится как поле**. Вычисляется при каждом запросе:

```sql
SELECT COUNT(*) FROM intervision_session_participants isp
JOIN intervision_sessions is ON isp.session_id = is.id
WHERE isp.user_id = ?
AND isp.confirmed_completion = true
AND is.status = 'completed'
AND is.starts_at >= NOW() - INTERVAL 30 DAY
```

Сравнивается с `intervision_settings.min_sessions`.

### Где используется

- Каталог психологов: `can_consult = false` → скрыт
- Дашборд психолога: показывает прогресс
- Записи на консультации: `can_consult = false` → новые невозможны

### Оптимизация

Если каталог начнёт тормозить — вынести в материализованный столбец или кэш с TTL. На старте — считать на лету.

### In scope (делаем)

- Заявка на создание постоянной группы + модерация
- Автогенерация сессий по расписанию (на месяц вперёд)

- Права создателя группы (управление составом, расписанием, закрытие)

- Видеосвязь через Jitsi (встроенный)
- Подтверждение сессии всеми участниками

- Напоминания за 24ч и 1ч
- Автоматический переход статусов сессий
- Динамическое вычисление can_consult
- Страница настроек в админке (min_sessions)

- Запись/трансляция сессий
- Чат внутри сессии (есть видеосвязь)
- Рейтинг/оценка участников интервизии
- Автоматический подбор группы
- Платные интервизии

## 13. Технические заметки

- **Jitsi:** комнаты создаются динамически по паттерну `intervision-session-{id}`. Настроить Jitsi Meet для авторизации через JWT, чтобы только участники сессии могли подключиться.
- **Автогенерация сессий:** Artisan-команда `intervision:generate-sessions`, запускается через Scheduler ежедневно. Для каждой активной группы — проверить, есть ли сессии на месяц вперёд, догенерировать недостающие.
- **Миграция данных:** `intervision_attendance` → `intervision_session_participants`. Для каждой записи в attendance с `attended = true` → создать запись с `confirmed_completion = true`. Запустить миграцию данных перед удалением таблицы.
- **Блокировка пересечений:** общая проверка для троек и интервизий. При записи на любой слот/сессию — проверить все слоты троек + все сессии интервизий.
- **Scheduler-задачи:** использовать `withoutOverlapping()` для предотвращения параллельного запуска.
- **Напоминания:** использовать Laravel Notifications. Пометить отправленные напоминания (добавить поле `reminder_24h_sent`, `reminder_1h_sent` в `intervision_sessions`), чтобы не дублировать при повторном запуске.
